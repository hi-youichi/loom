# Loom TUI 界面设计与交互说明

> 本文档描述 Loom TUI（终端用户界面）的设计方案和交互规范，不涉及技术实现细节。

---

## 一、界面概览

Loom TUI 提供一个可视化的 Agent 管理和监控界面，支持多 Agent 并发执行、实时进度追踪和多轮对话。

### 启动方式

```bash
loom tui
```

---

## 二、界面布局

### 整体结构

界面分为四个主要区域，从上到下依次为：

```
┌─────────────────────────────────────────────────────────┐
│                    标题栏                               │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                   Agent 列表区                          │
│              （显示所有 Agent 状态）                     │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                   消息历史区                            │
│              （显示用户输入历史）                        │
├─────────────────────────────────────────────────────────┤
│                    输入框                               │
│              （用户输入 prompt）                        │
└─────────────────────────────────────────────────────────┘
```

---

## 三、各区域详细设计

### 3.1 标题栏

**位置：** 界面顶部  
**高度：** 3 行  
**内容：** 居中显示 "Loom - Concurrent Agents Dashboard"  
**样式：** 青色粗体文字

---

### 3.2 Agent 列表区

**位置：** 中间主体区域  
**高度：** 自适应（占据剩余空间）  
**功能：** 显示所有 Agent 的状态卡片

#### 空状态

当没有 Agent 运行时：

```
┌─ Agents ────────────────────────────────────┐
│                                             │
│           No agents running                 │
│                                             │
└─────────────────────────────────────────────┘
```

- 显示黄色居中提示文字："No agents running"

#### Agent 卡片

每个 Agent 显示为一个独立卡片，高度固定 5 行：

```
┌─ Agent: agent-001 ──────────────────────────┐
│ CodeAnalyzer - ● Running                    │
│ Task: 重构代码模块                           │
│ Node: reasoning                             │
│ Progress: 正在分析依赖关系...                │
└─────────────────────────────────────────────┘
```

**卡片内容说明：**

| 显示项 | 说明 | 显示时机 |
|--------|------|---------|
| Agent ID | 卡片标题栏 | 始终显示 |
| 名称 | Agent 的名称 | 始终显示 |
| 状态图标 + 文字 | 运行状态 | 始终显示 |
| Task | 任务描述 | 始终显示 |
| Node | 当前执行节点 | Running 状态 |
| Progress | 进度消息 | 有更新时 |
| Result | 执行结果 | Completed 状态 |
| Error | 错误信息 | Error 状态 |

---

### 3.3 消息历史区

**位置：** Agent 列表下方  
**高度：** 固定 5 行  
**功能：** 显示用户输入的所有 prompt 历史

```
┌─ Messages ──────────────────────────────────┐
│ > 重构代码模块                               │
│ > 生成单元测试                               │
│ > 添加错误处理                               │
│ > 优化性能                                   │
│ > 更新文档                                   │
└─────────────────────────────────────────────┘
```

**交互规则：**
- 每次用户提交 prompt，新消息追加到列表
- 超过 5 条时，显示最新的 5 条
- 长消息自动换行

---

### 3.4 输入框

**位置：** 界面底部  
**高度：** 固定 3 行  
**功能：** 用户输入 prompt 的区域

#### Normal 模式（查看模式）

```
┌─ Input (press 'i' to edit, 'q' to quit) ────┐
│                                             │
└─────────────────────────────────────────────┘
```

- 边框颜色：灰色
- 输入区不可编辑
- 提示文字：操作说明

#### Editing 模式（编辑模式）

```
┌─ Input (press 'Enter' to submit, 'Esc' to cancel) ┐
│ 重构 src/lib.rs_|                                 │
└───────────────────────────────────────────────────┘
```

- 边框颜色：黄色
- 输入文字颜色：绿色
- 显示光标
- 提示文字：操作说明

## 四、状态系统

### 4.1 Agent 状态

Agent 有三种状态，通过颜色和图标区分：

| 状态 | 图标 | 颜色 | 说明 |
|------|------|------|------|
| Running | ● | 黄色 | Agent 正在执行任务 |
| Completed | ✓ | 绿色 | Agent 已成功完成任务 |
| Error | ✗ | 红色 | Agent 执行出错 |

### 4.2 输入模式

输入框有两种模式：

| 模式 | 边框颜色 | 功能 |
|------|---------|------|
| Normal | 灰色 | 查看状态，不可输入 |
| Editing | 黄色 | 可输入和编辑文本 |

## 五、交互设计

### 5.1 键盘快捷键

#### 全局快捷键

| 按键 | 功能 | 适用模式 |
|------|------|---------|
| `Ctrl + C` | 强制退出 TUI | 任意模式 |
| `q` | 退出 TUI | Normal |
| `Esc` | 退出 TUI | Normal |
| `i` | 进入编辑模式 | Normal |

#### 编辑模式快捷键

| 按键 | 功能 |
|------|------|
| `Enter` | 提交 prompt，返回 Normal 模式 |
| `Esc` | 取消输入，清空文本，返回 Normal 模式 |
| `Backspace` | 删除前一个字符 |
| `任意字符` | 输入到文本框 |

## 六、视觉设计规范

### 6.1 颜色方案

| 元素 | 颜色 | 用途 |
|------|------|------|
| 标题文字 | 青色 + 粗体 | 突出标题 |
| Messages 边框 | 蓝色 | 区分消息区域 |
| Agent Running 边框/图标 | 黄色 | 表示进行中 |
| Agent Completed 边框/图标 | 绿色 | 表示成功 |
| Agent Error 边框/图标 | 红色 | 表示失败 |
| Agent 名称 | 粗体 | 突出显示 |
| 字段标签（Task/Node/Progress 等） | 灰色 | 辅助信息 |
| 输入框边框（Normal） | 灰色 | 不可编辑状态 |
| 输入框边框（Editing） | 黄色 | 可编辑状态 |
| 输入文字（Editing） | 绿色 | 用户输入 |

### 6.2 图标

| 状态 | 图标 | Unicode |
|------|------|---------|
| Running | ● | U+25CF |
| Completed | ✓ | U+2713 |
| Error | ✗ | U+2717 |

### 6.3 布局尺寸

| 区域 | 高度 |
|------|------|
| 标题栏 | 固定 3 行 |
| Agent 列表 | 自适应（占剩余空间） |
| 每个 Agent 卡片 | 固定 5 行 |
| 消息历史 | 固定 5 行 |
| 输入框 | 固定 3 行 |
| 外边距 | 1 字符 |

## 七、用户操作指南

### 7.1 基本操作流程

1. **启动 TUI**
   ```bash
   loom tui
   ```

2. **输入任务**
   - 按 `i` 进入编辑模式
   - 输入 prompt（例如："重构代码模块"）
   - 按 `Enter` 提交

3. **查看进度**
   - 在 Agent 列表区查看实时进度
   - 进度消息会自动更新

4. **查看结果**
   - Agent 完成后，状态变为 ✓ Completed
   - Result 字段显示执行结果

5. **继续对话**
   - 重复步骤 2-4，可继续提交新任务
   - 消息历史会记录所有输入

6. **退出 TUI**
   - 按 `q` 或 `Esc` 退出

---

### 7.2 注意事项

- **并发任务：** 可以在 Agent 运行时提交新任务，多个 Agent 会并发执行
- **取消输入：** 在编辑模式下按 `Esc` 可取消当前输入
- **强制退出：** 按 `Ctrl + C` 可立即退出（不等待 Agent 完成）
- **长消息：** 消息历史和 Agent 卡片都支持自动换行，长文本会完整显示

## 八、界面状态流转

### 8.1 输入模式流转

```
         按 'i'
Normal ──────────▶ Editing
   ▲                   │
   │                   │
   └───────────────────┘
         按 'Esc' 或 'Enter'
```

### 8.2 Agent 状态流转

```
         启动 Agent
           │
           ▼
       Running ──────────▶ Completed
           │                   ▲
           │                   │
           └───▶ Error ────────┘
                (失败)
```

## 九、设计原则

### 9.1 简洁直观

- 状态通过颜色和图标一目了然
- 操作提示始终可见
- 布局清晰，功能区域分明

### 9.2 实时反馈

- Agent 进度实时更新
- 状态变化立即显示
- 用户输入即时响应

### 9.3 高效操作

- 最小化按键操作（`i` → 输入 → `Enter`）
- 支持快速退出（`q` 或 `Esc`）
- 无需记忆复杂命令

### 9.4 容错性

- 支持取消输入（`Esc`）
- 错误状态清晰显示
- 强制退出选项（`Ctrl + C`）

## 十、未来扩展（规划）

以下功能在当前设计中预留了扩展空间：

### 10.1 Agent 列表滚动

- 当 Agent 数量超过可视区域时，支持上下滚动
- 显示滚动指示器

### 10.2 消息历史滚动

- 支持查看所有历史消息
- 可选的搜索/过滤功能

### 10.3 Agent 详细日志

- 点击/选中 Agent 卡片查看详细执行日志
- 支持日志搜索和过滤

### 10.4 任务管理

- 支持取消运行中的 Agent
- 支持重试失败的 Agent
- 支持清空已完成的 Agent

### 10.5 主题定制

- 支持选择不同的颜色主题
- 自定义快捷键

# Session 会话区设计

本文档详细描述 Session 会话区的消息类型、视觉设计和交互方案。

---

## 1. 消息类型定义

### 1.1 消息分类

Session 会话包含以下消息类型：

| 类型 | 标识 | 来源 | 重要性 |
|------|------|------|--------|
| **用户消息** | `user` | 用户输入 | 高 |
| **助手消息** | `assistant` | AI 回复 | 高 |
| **思考过程** | `think` | AI 推理 | 中（可折叠） |
| **工具调用** | `tool` | 工具执行 | 中（可折叠） |
| **系统消息** | `system` | 系统状态 | 低 |

### 1.2 消息结构

每条消息包含：
- **时间戳**：消息产生时间
- **类型标识**：用于视觉区分
- **内容**：消息主体
- **元数据**：可选的附加信息（如工具参数、执行时长等）

### 2.2 消息类型视觉规范

#### 用户消息

```
[14:23:45] 👤 User
┌────────────────────────────────────────────────────────┐
│ 重构这个代码模块，优化性能                              │
└────────────────────────────────────────────────────────┘
```

- **图标**：👤 (或 `>`)
- **颜色**：青色边框，白色文字
- **背景**：无背景色
- **标识**：`User`

#### 助手消息

```
[14:23:47] 🤖 Assistant
┌────────────────────────────────────────────────────────┐
│ 我来帮你重构这个代码模块。首先让我分析一下当前的代码  │
│ 结构和性能瓶颈...                                      │
└────────────────────────────────────────────────────────┘
```

- **图标**：🤖 (或 `AI`)
- **颜色**：绿色边框，白色文字
- **背景**：无背景色
- **标识**：`Assistant`

#### 思考过程（可折叠）

**折叠状态：**
```
[14:23:48] 💭 Think ▶ [+展开]
```

**展开状态：**
```
[14:23:48] 💭 Think ▼ [-折叠]
┌────────────────────────────────────────────────────────┐
│ 用户希望重构代码模块并优化性能。我需要：                │ │
│ 1. 先读取代码文件了解当前结构                          │ │
│ 2. 识别性能瓶颈                                        │ │
│ 3. 提出重构方案...                                     │ │
└────────────────────────────────────────────────────────┘
```

- **图标**：💭 (或 `?`)
- **颜色**：黄色边框，灰色文字
- **背景**：淡灰色背景（可选）
- **标识**：`Think`
- **交互**：默认折叠，按 `Enter` 或 `Space` 展开/折叠

#### 工具调用（可折叠）

**折叠状态（成功）：**
```
[14:23:50] 🔧 Tool: read_file ✓
┌────────────────────────────────────────────────────────┐
│ 📄 Path: src/lib.rs  |  ⏱ 23ms  |  [Enter 展开]       │
└────────────────────────────────────────────────────────┘
```

**展开状态：**
```
[14:23:50] 🔧 Tool: read_file ✓
┌────────────────────────────────────────────────────────┐
│ 📄 Path: src/lib.rs                                    │ │
│ ⏱ Duration: 23ms                                       │ │
│ ────────────────────────────────────────────────────── │ │
│ Output:                                                 │ │
│   1 │ pub fn calculate(x: i32) -> i32 {                │ │
│   2 │     let mut result = 0;                          │ │
│   3 │     for i in 0..x {                              │ │
│   4 │         result += i * i;                         │ │
│   5 │     }                                            │ │
│   ...                                                  │ │
└────────────────────────────────────────────────────────┘
```

**错误状态：**
```
[14:23:50] 🔧 Tool: read_file ✗
┌────────────────────────────────────────────────────────┐
│ 📄 Path: src/lib.rs  |  ⏱ 5ms  |  [Enter 查看错误]    │
└────────────────────────────────────────────────────────┘
```

- **图标**：🔧 (或 `T`)
- **颜色**：蓝色边框（成功）/ 红色边框（失败）
- **标识**：`Tool: {tool_name}`
- **状态**：✓ 成功 / ✗ 失败
- **元数据**：工具参数、执行时长、输出内容
- **交互**：默认折叠，按 `Enter` 或 `Space` 展开/折叠

#### 系统消息

```
[14:23:55] ⚠ System
┌────────────────────────────────────────────────────────┐
│ Agent 遇到错误，正在重试...                             │
└────────────────────────────────────────────────────────┘
```

- **图标**：⚠ (或 `!`)
- **颜色**：红色边框，黄色文字
- **标识**：`System`
- **类型**：错误、警告、状态更新

## 3. 交互设计

### 3.1 滚动和导航

| 按键 | 功能 |
|------|------|
| `↑` / `k` | 向上滚动一行 |
| `↓` / `j` | 向下滚动一行 |
| `Page Up` | 向上滚动一页 |
| `Page Down` | 向下滚动一页 |
| `Home` / `g` | 跳转到第一条消息 |
| `End` / `G` | 跳转到最后一条消息 |
| `Ctrl+U` | 向上滚动半页 |
| `Ctrl+D` | 向下滚动半页 |

### 3.2 消息交互

| 按键 | 功能 |
|------|------|
| `Enter` / `Space` | 展开/折叠当前选中的消息 |
| `Tab` | 在可折叠消息之间跳转 |
| `Shift+Tab` | 反向跳转 |
| `y` | 复制当前消息内容到剪贴板 |
| `f` | 过滤特定类型的消息 |

### 3.3 自动滚动

- **新消息时**：自动滚动到底部（如果在底部）
- **浏览历史时**：暂停自动滚动，显示新消息提示
- **新消息提示**：`[3 条新消息] - 按 End 跳转`

### 3.4 消息过滤

按 `f` 打开过滤菜单：

```
┌─ 消息过滤 ──────────────────────────────────┐
│ [✓] 用户消息 (User)                         │
│ [✓] 助手消息 (Assistant)                    │
│ [✓] 思考过程 (Think)                        │
│ [✓] 工具调用 (Tool)                         │
│ [ ] 系统消息 (System)                       │
│                                              │
│ [Enter] 应用  [Esc] 取消                    │
└──────────────────────────────────────────────┘
```

## 4. 特殊场景处理

### 4.1 长消息处理

**策略：**
1. **单行截断**：超过宽度的行显示 `...`
2. **折叠长消息**：超过 10 行的消息默认折叠，显示前 3 行和后 2 行
3. **代码块**：检测代码块并保持格式，提供水平滚动

**示例：**
```
[14:24:00] 🤖 Assistant
┌────────────────────────────────────────────────────────┐
│ 这是一个很长的回复，包含了详细的代码说明...            │ │
│                                                          │ │
│ ```rust                                                 │ │
│ fn example() {                                          │ │
│     // 代码内容...                                      │ │
│ }                                                       │ │
│ ```                                                     │ │
│                                                          │ │
│ ... [还有 15 行已折叠，按 Enter 展开] ...               │ │
│                                                          │ │
│ 总结：以上代码实现了...                                 │ │
└────────────────────────────────────────────────────────┘
```

### 4.2 代码块处理

- 检测 ````` 语言标识符
- 使用语法高亮（如果终端支持）
- 长代码块可折叠
- 显示行号（可选）

### 4.3 工具批量调用

当连续调用多个工具时，使用折叠组：

```
[14:24:00] 🔧 3 Tools Called ▶ [+展开]
┌────────────────────────────────────────────────────────┐
│ • read_file (23ms) ✓                                   │ │
│ • grep (45ms) ✓                                         │ │
│ • edit (12ms) ✓                                         │ │
└────────────────────────────────────────────────────────┘
```

### 4.4 错误处理

工具失败时显示详细错误：

```
[14:24:00] 🔧 Tool: read_file ✗
┌────────────────────────────────────────────────────────┐
│ 📄 Path: src/lib.rs                                    │ │
│ ⏱ Duration: 5ms                                        │ │
│ ────────────────────────────────────────────────────── │ │
│ ❌ Error: File not found                                │ │
│   The file "src/lib.rs" does not exist.                │ │
│                                                          │ │
│ [Retry] [Skip] [Abort]                                  │ │
└────────────────────────────────────────────────────────┘
```

## 5. 性能优化

### 5.1 虚拟滚动

- 只渲染可见区域的消息
- 缓存消息渲染结果
- 动态计算滚动位置

### 5.2 消息限制

- 保留最近 1000 条消息
- 超过后删除最旧的消息
- 提供"加载更多"功能（如果支持历史消息）

### 5.3 增量更新

- 只重新渲染变化的消息
- 批量更新消息（避免频繁重绘）

## 6. 状态栏设计

在 Session 会话区底部显示状态栏：

```
┌─ Session: session-abc123 ────────────────────────────────────┐
│ ... (消息内容) ...                                          │
├──────────────────────────────────────────────────────────────┤
│ 💬 42 messages  |  🔧 15 tools  |  ⚠ 2 errors  |  ↓ Auto   │
└──────────────────────────────────────────────────────────────┘
```

状态栏信息：
- 总消息数
- 工具调用数
- 错误数
- 滚动状态（Auto/Manual）
- 当前过滤状态（如果有）

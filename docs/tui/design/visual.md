# Loom TUI 视觉设计文档

本文档描述 Loom TUI（终端用户界面）的视觉设计，不涉及交互和技术实现细节。

---

## 1. 界面概览

### 1.1 整体布局

Loom TUI 采用三区域布局，包含顶部标题栏和下方两栏布局：

```
┌─────────────────────────────────────────────────────────────────────┐
│                          标题栏                                     │
├────────────────────────────────┬────────────────────────────────────┤
│                                │                                    │
│      Agent 网格区              │         Session 会话区             │
│   （显示所有 Agent 的状态）     │    （显示当前 Session 的对话）     │
│                                │                                    │
│                                │                                    │
├────────────────────────────────┴────────────────────────────────────┤
│                          输入框                                     │
│                 （用户输入 prompt 的区域）                          │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 各区域说明

| 区域 | 功能 | 尺寸 |
|------|------|------|
| 标题栏 | 显示应用名称和状态 | 固定 3 行 |
| Agent 网格区 | 显示所有 Agent 的状态卡片 | 占 40% 宽度，高度自适应 |
| Session 会话区 | 显示当前 Session 的对话历史 | 占 60% 宽度，高度自适应 |
| 输入框 | 用户输入新的 prompt | 固定 3 行 |

## 2. Agent 网格区设计

### 2.1 网格布局

Agent 网格区采用响应式网格布局，根据可用空间自动调整列数：

```
┌─ Agent Grid ─────────────────────┐
│  ┌──────────┐  ┌──────────┐     │
│  │ Agent 1  │  │ Agent 2  │     │
│  └──────────┘  └──────────┘     │
│  ┌──────────┐  ┌──────────┐     │
│  │ Agent 3  │  │ Agent 4  │     │
│  └──────────┘  └──────────┘     │
│  ┌──────────┐                   │
│  │ Agent 5  │                   │
│  └──────────┘                   │
└──────────────────────────────────┘
```

**布局规则：**
- 最小卡片宽度：20 字符
- 卡片间距：1 字符
- 自动计算列数：`floor(可用宽度 / (最小宽度 + 间距))`
- 最多显示 3 列（避免卡片过窄）

### 2.2 Agent 卡片结构

每个 Agent 以独立卡片形式显示，包含以下信息：

```
┌─ Agent: abc123 ────────────┐
│ CodeAnalyzer - ● Running   │  ← 名称 + 状态
│ Task: 重构代码模块          │  ← 任务描述
│ Node: reasoning            │  ← 当前节点
│ Progress: 分析依赖中...     │  ← 进度消息
└────────────────────────────┘
```

### 2.3 状态类型

Agent 有三种状态，每种状态有不同的视觉表现：

#### 运行中 (Running)

```
┌─ Agent: agent-001 ────────────┐
│ CodeAnalyzer - ● Running      │  ← 黄色图标和边框
│ Task: 重构代码模块             │
│ Node: reasoning               │
│ Progress: 正在分析依赖关系...  │
└───────────────────────────────┘
```

- **图标**: ● （实心圆点）
- **颜色**: 黄色
- **边框**: 黄色
- **显示内容**: 名称、任务、当前节点、进度消息

#### 已完成 (Completed)

```
┌─ Agent: agent-001 ────────────┐
│ CodeAnalyzer - ✓ Completed    │  ← 绿色图标和边框
│ Task: 重构代码模块             │
│ Result: 重构完成，优化了 3 个函数│
└───────────────────────────────┘
```

- **图标**: ✓ （对勾）
- **颜色**: 绿色
- **边框**: 绿色
- **显示内容**: 名称、任务、执行结果

#### 出错 (Error)

```
┌─ Agent: agent-001 ────────────┐
│ CodeAnalyzer - ✗ Error        │  ← 红色图标和边框
│ Task: 重构代码模块             │
│ Error: 文件不存在: src/lib.rs  │
└───────────────────────────────┘
```

- **图标**: ✗ （叉号）
- **颜色**: 红色
- **边框**: 红色
- **显示内容**: 名称、任务、错误信息

### 2.4 卡片尺寸

- **最小宽度**: 20 字符
- **最大宽度**: 40 字符
- **固定高度**: 5 行
- **内边距**: 1 字符

## 3. Session 会话区设计

Session 会话区是 TUI 的核心交互区域，用于显示 Agent 的完整对话历史，包括用户输入、AI 回复、思考过程、工具调用等。

### 3.1 消息类型概览

| 消息类型 | 图标 | 默认状态 | 说明 |
|---------|------|---------|------|
| 用户消息 | 👤 | 始终展开 | 用户的输入 prompt |
| 助手消息 | 🤖 | 始终展开 | AI 的回复内容 |
| 思考过程 | 💭 | 默认折叠 | AI 的内部推理过程 |
| 工具调用 | 🔧 | 默认折叠 | 工具名称、参数、结果 |
| 系统消息 | ℹ️/⚠️/❌ | 始终展开 | 系统信息、警告、错误 |

### 3.2 用户消息

```
┌─────────────────────────────────────────────────┐
│ 👤 You                                          │
│ ─────────────────────────────────────────────── │
│ 帮我重构 src/lib.rs 文件，优化性能并提高可读性  │
└─────────────────────────────────────────────────┘
```

**样式设计：**
- 背景色：深蓝色 (`#1e3a5f`)
- 文字色：白色
- 边框：蓝色
- 前缀：`👤 You`
- 分隔线：`─` 字符

### 3.3 助手消息

```
┌─────────────────────────────────────────────────┐
│ 🤖 Assistant                                    │
│ ─────────────────────────────────────────────── │
│ 我已经完成了代码重构。主要改进包括：             │
│                                                 │
│ 1. 提取了重复逻辑到独立函数                     │
│ 2. 优化了算法复杂度从 O(n²) 到 O(n log n)      │
│ 3. 添加了必要的注释和文档                       │
│                                                 │
│ 重构后的代码更加清晰和高效。                    │
└─────────────────────────────────────────────────┘
```

**样式设计：**
- 背景色：深灰色 (`#2d2d2d`)
- 文字色：绿色 (`#4ade80`)
- 边框：绿色
- 前缀：`🤖 Assistant`
- 支持 Markdown 基础格式（代码块、列表等）

### 3.4 思考过程

#### 折叠状态（默认）

```
┌─────────────────────────────────────────────────┐
│ 💭 Thinking (Click to expand)          [▶]      │
└─────────────────────────────────────────────────┘
```

#### 展开状态

```
┌─────────────────────────────────────────────────┐
│ 💭 Thinking                               [▼]   │
│ ─────────────────────────────────────────────── │
│ 首先需要分析函数的结构和依赖关系...             │
│                                                 │
│ 1. 检查函数的输入参数和返回值                   │
│ 2. 识别可能的性能瓶颈                           │
│ 3. 考虑重构的方案和影响                         │
│                                                 │
│ 看起来可以从提取公共逻辑开始...                 │
└─────────────────────────────────────────────────┘
```

**样式设计：**
- 背景色：深紫色 (`#2d1b4e`)
- 文字色：浅紫色 (`#c4b5fd`)
- 边框：紫色
- 前缀：`💭 Thinking`
- 折叠/展开指示器：`[▶]` / `[▼]`
- 斜体显示思考内容

**交互：**
- 点击或按 `Enter` 切换折叠/展开
- 按 `t` 全局切换所有思考块的显示

### 3.5 工具调用

#### 折叠状态（默认）

```

│ 🔧 read_file              ✓ 0.23s      [▶]      │
└─────────────────────────────────────────────────┘
```

#### 展开状态（成功）

```
┌─────────────────────────────────────────────────┐

│ Parameters:                                     │
│   path: "src/lib.rs"                           │
│   encoding: "utf-8"                            │
│                                                 │
│ Result:                                         │
│   1 │ use std::collections::HashMap;           │

│   3 │                                          │

│   5 │     // TODO: optimize this function      │
│   ... (显示前 50 行)                            │

```

#### 展开状态（进行中）

```
┌─────────────────────────────────────────────────┐
│ 🔧 write_file             ⏳ Running    [▼]      │
│ ─────────────────────────────────────────────── │
│ Parameters:                                     │
│   path: "src/lib.rs"                           │
│   content: "..." (2341 bytes)                  │
│                                                 │

```

#### 展开状态（失败）

```
┌─────────────────────────────────────────────────┐
│ 🔧 write_file             ✗ Error      [▼]      │
│ ─────────────────────────────────────────────── │
│ Parameters:                                     │

│                                                 │
│ Error:                                          │

```

**样式设计：**
- 背景色：深青色 (`#0d3d3d`)
- 文字色：青色 (`#22d3ee`)
- 边框：青色
- 前缀：`🔧 <tool_name>`
- 状态图标：
  - ✓ 成功（绿色）
  - ⏳ 进行中（黄色，带动画）
  - ✗ 失败（红色）
- 执行时间显示（秒）

**参数和结果格式化：**
- 使用缩进显示 JSON/结构化数据
- 代码块使用语法高亮
- 长内容自动截断，显示行数统计

### 3.8 滚动和导航

**自动滚动：**
- 新消息到达时自动滚动到底部
- 用户手动滚动时暂停自动滚动
- 底部显示提示：`▼ 滚动查看更多 (12 条新消息)...`

**滚动指示器：**
```
┌─ Session: main-session ─────────────────────┐
│ [14:23:45] Message 1...                     │
│ [14:24:12] Message 2...                     │
│ ...                                         │
│ ▲ 上面还有 23 条消息                        │
│ [14:35:01] Current message...               │
│ ▼ 下面还有 5 条消息                         │
└─────────────────────────────────────────────┘
```

**分页滚动：**
- `Ctrl+D`: 向下翻半页
- `Ctrl+U`: 向上翻半页
- `Ctrl+F`: 向下翻整页
- `Ctrl+B`: 向上翻整页

**快速跳转：**
- `gg`: 跳转到第一条消息
- `G`: 跳转到最后一条消息
- `/`: 搜索消息内容

## 4. 输入框设计

### 4.1 布局

```
┌─ Input ────────────────────────────────────────┐
│ > 请帮我分析这段代码的性能瓶颈                 █│
│                                                │
│ Input (press 'i' to edit, 'Enter' to submit)  │
└────────────────────────────────────────────────┘
```

### 4.2 状态样式

#### Normal 模式（查看模式）

- 边框: 灰色
- 文字: 灰色
- 无光标
- 提示: `Input (press 'i' to edit, 'q' to quit)`

#### Editing 模式（编辑模式）

- 边框: 黄色
- 文字: 绿色
- 显示光标
- 提示: `Input (press 'Enter' to submit, 'Esc' to cancel)`

## 7. 响应式设计

### 7.1 尺寸适配

| 终端宽度 | Agent 网格 | Session 区域 |
|---------|-----------|-------------|
| > 120 列 | 3 列网格 | 完整显示 |
| 80-120 列 | 2 列网格 | 正常显示 |
| 60-80 列 | 1 列网格 | 紧凑显示 |
| < 60 列 | 显示警告 | 建议扩大终端 |

### 7.2 高度适配

- 最小高度：20 行
- 低于 20 行时显示警告
- Agent 网格区最少显示 2 行卡片
- Session 区最少显示 3 条消息
